# The ARIADNE lab

The lab is where detections meet *real activity* instead of hand-written JSON.
Nothing here touches a production system; every scenario operates on disposable,
locally-created decoy resources and cleans up after itself.

## Real-activity scenario

```bash
bash lab/scenario_actions/source_code_exfiltration.sh
```

This performs genuine benign actions and then verifies the detection fires on the
telemetry they produce:

1. creates several decoy git repositories with synthetic "restricted" files and
   honeytokens,
2. creates a staging directory standing in for removable media,
3. clones the repositories with real `git`, searches for credential-pattern
   files, builds an archive (`gpg` if present, otherwise `tar`), and copies it to
   staging,
4. writes and deletes a throwaway shell-history file and drops a
   "telemetry stopped" marker,
5. records every action to `actions.jsonl`,
6. maps those actions to normalized events (`emit_events.py`) and replays them
   through the shipped detection pack, asserting `ARI-IR-0042` opens a case.

It exits non-zero if the detection fails to fire, so it doubles as an
end-to-end smoke test of the real-activity path.

## Optional backends

The pure-Python core needs no infrastructure. For the high-volume and
evidence-storage paths:

```bash
docker compose -f lab/docker/docker-compose.yml up
```

- **ClickHouse** — runs the ClickHouse-compiled detections over large volumes.
- **PostgreSQL** — cases, identities, and rule metadata.
- **MinIO** — immutable evidence objects (see `ariadne.events.provenance`).
- **console** — the FastAPI investigation UI on `:8000`.

## Synthetic enterprise

The benign background telemetry every scenario layers its incident on top of is
generated by `ariadne.lab.synthetic.SyntheticEnterprise` — a seeded population of
employees and devices emitting logins, processes, clones, DNS, and network flows.
`scenarios/*/build.py` use it to regenerate their event fixtures deterministically.

## Disposability

The `decoy_repositories/` and `synthetic_enterprise/` directories are generation
targets, not checked-in data. Everything the scenario script creates lives under a
`mktemp` directory removed on exit. No real credentials, no external victim, no
network egress.
