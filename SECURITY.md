# Security Policy

## Authorized, defensive use

ARIADNE is a defensive detection-engineering framework. It is intended for
authorized insider-risk and threat-detection programs operating with appropriate
organizational, legal, and HR oversight, and for security research and education.

It analyzes telemetry an organization already collects. It is **not** a
surveillance product, and it deliberately favors explainability over an opaque
score precisely so that its output supports a fair, contestable review rather
than an automatic accusation. Every firing reports the minimal decisive evidence,
the competing benign explanations, and the evidence missing from the record. See
[docs/privacy-and-ethics.md](docs/privacy-and-ethics.md).

Do not deploy ARIADNE against systems or people you are not authorized to
monitor.

## Scope and trust assumptions

ARIADNE's reference engine is the deterministic ground truth. Its trust boundary
is the **normalized event**: collectors and the normalizer are assumed to deliver
faithful, attributed records. Tampering upstream of normalization (forged
telemetry, identity spoofing the resolver cannot untangle) is out of scope for the
engine and is addressed by the collection layer and by
[docs/threat-model.md](docs/threat-model.md).

## Reporting a vulnerability

If you discover a security issue in ARIADNE itself — for example a way to make the
engine produce a non-deterministic result, a denial-of-service in evaluation, or a
parsing flaw in a collector — please report it privately rather than opening a
public issue:

1. Open a GitHub security advisory on the repository, or
2. Contact the maintainers through the address listed on the repository profile.

Please include a minimal reproduction (a small event stream and detection, or a
failing property) and the observed versus expected behaviour. We aim to
acknowledge reports within a few days.

## Handling test and scenario data

All telemetry in this repository is synthetic and generated by seeded builders
(`scenarios/*/build.py`, `ariadne.lab.synthetic`). It contains no real
credentials, hosts, or people. The lab scenario actions operate only on disposable
local decoy resources and clean up after themselves; never point them at
production systems.
